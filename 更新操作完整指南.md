### 配置定时任务（方法2：PowerShell）

```powershell
# 定义任务信息
$taskName = "云行业HR洞察-自动更新"
$scriptPath = "C:\Users\赵梓菡\WorkBuddy\Claw\netlify-deploy\schedule_update.bat"
$workDir = "C:\Users\赵梓菡\WorkBuddy\Claw\netlify-deploy"
$time = "18:00"

# 创建触发器
$trigger = New-ScheduledTaskTrigger -Daily -At $time

# 创建操作
$action = New-ScheduledTaskAction `
    -Execute $scriptPath `
    -WorkingDirectory $workDir

# 设置任务参数
$settings = New-ScheduledTaskSettingsSet `
    -AllowStartIfOnBatteries `
    -DontStopIfGoingOnBatteries `
    -StartWhenAvailable

# 创建任务
Register-ScheduledTask `
    -TaskName $taskName `
    -Trigger $trigger `
    -Action $action `
    -Settings $settings `
    -Description "云行业HR洞察 - 每天自动抓取新闻并部署"

Write-Host "✅ 定时任务创建成功！" -ForegroundColor Green
```

## 4. 时间范围筛选

### 4.1 理解时间范围

| 视图 | 时间范围 | 说明 |
|------|----------|------|
| **今日观察** | 当天（如 2026-03-16） | 只显示当天发布的新闻 |
| **本周观察** | 周一到周日（如 3.9-3.15） | 只显示本周发布的新闻 |

---

### 4.2 手动筛选新闻

**方法1：查看新闻日期**

每条新闻都有发布时间：
```html
<div class="meta">📅 发布时间: 今日 10:00 | 📌 信息来源: 华为云官网</div>
```

或者：
```html
<div class="source">
    <p>📅 2026-03-10 | 📌 科锐国际 | 🔗 <a href="#">查看详情</a></p>
</div>
```

**方法2：删除不符合时间范围的新闻**

如删除 2025-12-03 的新闻：
1. 找到该新闻的 `<div class="news-card">` 标签
2. 删除整个 `<div>` 及其内容
3. 保存文件

### Q2: 新闻抓取失败怎么办？

**检查步骤：**

1. **检查网络连接**
   ```powershell
   ping www.google.com
   ```

2. **查看日志文件**
   ```powershell
   type auto_update.log
   ```

3. **更新新闻源配置**

   编辑 `auto_update.py`，修改 `NEWS_SOURCES`：

   ```python
   NEWS_SOURCES = [
       {
           'name': '新新闻源',
           'url': 'https://example.com/news',
           'selector': '.news-item',
           'title_selector': '.title',
           'date_selector': '.date',
           'link_selector': 'a'
       }
   ]
   ```

4. **添加 User-Agent 或代理**

   ```python
   headers = {
       'User-Agent': 'Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36',
       'Proxy': 'http://proxy.example.com:8080'  # 如需要
   }
   ```

---

### Q3: Git 推送失败怎么办？

**检查步骤：**

1. **验证 Git 配置**
   ```powershell
   git config --global user.name
   git config --global user.email
   ```

2. **检查远程仓库**
   ```powershell
   git remote -v
   ```

3. **重新推送**
   ```powershell
   git pull origin main
   git push origin main
   ```

4. **使用 Personal Access Token**

   如果推送失败，可能需要用 Token 代替密码：
   1. 访问 https://github.com/settings/tokens
   2. 生成新 Token，勾选 `repo` 权限
   3. 推送时输入 Token 作为密码

### Q4: 如何回滚到旧版本？

**方法1：使用 Git 命令**
```powershell
# 查看提交历史
git log --oneline

# 回滚到指定版本
git reset --hard <commit-id>

# 强制推送（谨慎使用）
git push -f
```

**方法2：在 Netlify 回滚**

1. 访问 Netlify 部署页面
2. 点击 **"Deploys"** 标签
3. 找到要回滚的版本
4. 点击 **"Publish deploy"** 旁边的三个点
5. 选择 **"Rollback to this deploy"**

---

### Q5: 网站没有更新怎么办？

**检查步骤：**

1. **检查 GitHub 提交**
   - 访问：https://github.com/lumoszhao/cloud-hr-insight
   - 查看最新提交时间

2. **检查 Netlify 部署状态**
   - 访问：https://app.netlify.com/

3. **清除浏览器缓存**
   - 按 `Ctrl + Shift + R` 强制刷新

4. **等待几分钟**
   - Netlify 部署需要 1-2 分钟

### Q6: 如何停止自动更新？

**方法1：禁用定时任务**
```powershell
Disable-ScheduledTask -TaskName "云行业HR洞察-自动更新"
```

**方法2：删除定时任务**
```powershell
Unregister-ScheduledTask -TaskName "云行业HR洞察-自动更新" -Confirm:$false
```

**方法3：停止批处理脚本中的循环**

编辑 `schedule_update.bat`，删除或注释掉：
```batch
REM timeout /t 32400 /nobreak > nul
REM goto loop
```

## 9. 最佳实践

### 日常维护

1. **定期检查日志**，确保脚本正常运行
2. **每周验证网站**，确认内容正确更新
3. **备份重要数据**，定期提交到 GitHub
4. **更新新闻源配置**，确保抓取准确性

### 内容管理

1. **手动编辑时**，先备份原文件
2. **修改后**，先本地预览再推送
3. **重要更新**，添加详细的 commit 描述
4. **定期归档**历史报告，便于回顾

### 安全建议

1. **不要**在代码中存储敏感信息（API 密钥、密码）
2. **使用**环境变量存储配置
3. **定期更新** Python 依赖库
4. **使用**强密码保护 GitHub 账号

## 10. 总结

现在你有两种更新方式：

| 方式 | 优点 | 缺点 |
|------|------|------|
| **手动更新** | 灵活可控，适合人工筛选 | 需要手动操作 |
| **自动更新** | 完全自动，定时运行 | 需要配置脚本 |

**推荐：**
- 日常使用**手动更新**（灵活性高）
- 配置**自动更新**作为补充（节省时间）
